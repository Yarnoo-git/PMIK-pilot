## 1. 소개

### 1.1 PMIK-pilot이란?

PMIK-pilot은 **직원 의견 조사(EOS) 데이터를 체계적으로 분석**하는 Python 기반 시스템입니다.

**주요 목적**:
- 📊 설문 응답 현황 분석
- 💡 직원 동기부여 및 저해 요인 파악
- 🎯 조직 개선을 위한 인사이트 도출
- 📈 데이터 기반 HR 의사결정 지원

---

### 1.2 주요 기능

이 시스템으로 다음 6가지 분석을 수행할 수 있습니다:

| 분석 유형 | 분석 내용 | 소요 시간 |
|----------|----------|----------|
| 🏢 **부서별 응답 분석** | 사업부/부서/팀별 응답률 현황 | ~5초 |
| ⏳ **근속기간별 분석** | 근속연수별 응답률 및 패턴 | ~5초 |
| 👔 **직급별 분석** | 직급별 응답률 및 특징 | ~5초 |
| 💪 **동기부여 요인 분석** | Q75: 업무 몰입 동기 요인 Top 5 | ~5초 |
| 🚧 **저해 요인 분석** | Q76: 업무 몰입 저해 요인 Top 5 | ~5초 |
| 🔄 **비교 분석** | Q75 vs Q76 근속연수별 비교 | ~10초 |

**전체 분석 소요 시간**: 약 **1분**

#### 무엇을 분석하나요?

**Q75 문항**: "귀하로 하여금 업무에 몰입하게 하는 요인 중에 가장 중요한 요인은 무엇입니까?"

- 사업부별 Top 3
- 직급별 Top 3
- 가장 인기 있는 조합 Top 10

#### 12개 선택지

1. 회사의 성장 가능성
2. 조직문화 및 분위기
3. 높은 금전적 보상 수준
4. 충분한 복리후생 지원
5. 다양한 업무 경험
6. 교육 프로그램
7. 적절한 권한위임
8. 성과, 능력에 따른 평가
9. 상사의 리더십
10. 일과 개인생활의 조화 (워라밸)

### 6.3 맞춤 분석 만들기

기존 스크립트를 참고하여 새로운 분석을 만들 수 있습니다.

#### 예제: 특정 문항(Q10) 분석 스크립트

**파일명**: `my_custom_analysis.py`

```python
import sqlite3
import pandas as pd
import sys

# UTF-8 인코딩 설정
sys.stdout.reconfigure(encoding='utf-8')

# 데이터베이스 연결
conn = sqlite3.connect('PMIK_2025.db')

print("=" * 80)
print("Q10 문항 분석")
print("=" * 80)

# Q10 문항 정보 조회
query_q10 = """
SELECT 문항, "선택(보기)"
FROM pmik_eos
WHERE "No." = 10.0
LIMIT 1
"""
q10_info = pd.read_sql_query(query_q10, conn)
print(f"\n[Q10 문항]")
print(f"{q10_info['문항'].iloc[0]}")
print(f"\n선택지: {q10_info['선택(보기)'].iloc[0]}")

# Q10 응답 분포 분석
query_distribution = """
SELECT
    r010 as score,
    COUNT(*) as count,
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM pmik_raw_data WHERE completed = 1 AND r010 IS NOT NULL), 1) as percentage
FROM pmik_raw_data
WHERE completed = 1 AND r010 IS NOT NULL
GROUP BY r010
ORDER BY r010 DESC
"""
df_dist = pd.read_sql_query(query_distribution, conn)

print("\n[응답 분포]")
score_labels = {5: '매우 그렇다', 4: '그렇다', 3: '보통', 2: '그렇지 않다', 1: '전혀 그렇지 않다'}
for _, row in df_dist.iterrows():
    score = int(row['score'])
    count = int(row['count'])
    pct = row['percentage']
    label = score_labels.get(score, '')
    bar = '█' * int(pct / 5)
    print(f"{score}점 ({label:10s}): [{bar:20s}] {count:3d}명 ({pct:5.1f}%)")

# 사업부별 평균 점수
query_by_unit = """
SELECT
    etc1 as business_unit,
    ROUND(AVG(r010), 2) as avg_score,
    COUNT(*) as respondents
FROM pmik_raw_data
WHERE completed = 1 AND r010 IS NOT NULL
GROUP BY etc1
ORDER BY avg_score DESC
"""
df_unit = pd.read_sql_query(query_by_unit, conn)

print("\n[사업부별 평균 점수]")
for _, row in df_unit.iterrows():
    unit = row['business_unit']
    avg = row['avg_score']
    cnt = int(row['respondents'])
    print(f"  {unit:10s}: {avg:.2f}점 ({cnt}명)")

# 전체 평균
overall_avg = df_dist['score'].dot(df_dist['count']) / df_dist['count'].sum()
print(f"\n[전체 평균 점수]: {overall_avg:.2f}점")

conn.close()
print("\n" + "=" * 80)
```

**실행**:
```bash
python my_custom_analysis.py
```

## 8. FAQ

### 8.1 가상환경을 왜 사용하나요?

**답변**:
가상환경은 프로젝트별로 독립적인 Python 패키지 환경을 만듭니다.

**장점**:
- 프로젝트 간 패키지 충돌 방지
- 시스템 Python 오염 방지
- 정확한 버전 관리
- 쉬운 배포 (requirements.txt)

**예시**:
- 프로젝트 A: pandas 1.5.0 사용
- 프로젝트 B: pandas 2.0.0 사용
- 가상환경 없이는 하나만 설치 가능
- 가상환경으로 각각 독립 관리

---

### 8.2 데이터를 어떻게 업데이트하나요?

**답변**:
Excel 파일을 수정한 후 데이터베이스에 재업로드해야 합니다.

**방법 1: Excel-uploader 사용 (권장)**

별도 프로젝트인 [Excel-uploader](https://github.com/Yarnoo-git/Excel-uploader)를 사용:
```bash
cd ../Excel-uploader
python upload.py ../PMIK-pilot/pmik_raw_data.xlsx "pmik_raw_data" ../PMIK-pilot/PMIK_2025.db
```

**방법 2: Python 스크립트 작성**

```python
import pandas as pd
import sqlite3

# Excel 파일 읽기
df = pd.read_excel('pmik_raw_data.xlsx')

# 데이터베이스 연결
conn = sqlite3.connect('PMIK_2025.db')

# 기존 테이블 삭제 후 재생성
df.to_sql('pmik_raw_data', conn, if_exists='replace', index=False)

conn.close()
print("데이터 업데이트 완료!")
```

#### 예제 2: 사업부별 평균 근속기간

```sql
SELECT
    m."Biz Unit." as business_unit,
    COUNT(*) as employee_count,
    AVG(
        CAST(SUBSTR(m.근속기간, 1, INSTR(m.근속기간, '년')-1) AS REAL) +
        CAST(SUBSTR(m.근속기간, INSTR(m.근속기간, '년')+2, INSTR(m.근속기간, '개월')-INSTR(m.근속기간, '년')-2) AS REAL) / 12.0
    ) as avg_tenure_years
FROM pmik_member m
GROUP BY m."Biz Unit."
ORDER BY avg_tenure_years DESC;
```

---

#### 예제 3: Q75 선택지별 빈도 (복수 선택 처리)

```sql
SELECT
    e.비고 as option_number,
    e."선택(보기)" as option_text,
    COUNT(*) as selection_count,
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM pmik_raw_data WHERE completed = 1), 1) as percentage
FROM pmik_raw_data r, pmik_eos e
WHERE r.completed = 1
    AND e."No." = 75.0
    AND (',' || REPLACE(r.r075, ' ', ',') || ',') LIKE ('%,' || e.비고 || ',%')
GROUP BY e.비고, e."선택(보기)"
ORDER BY selection_count DESC;
```

**설명**:
- `REPLACE(r.r075, ' ', ',')`: "4 11 10" → "4,11,10"
- `',' || ... || ','`: ",4,11,10,"
- `LIKE ('%,' || e.비고 || ',%')`: ",4," 패턴 매칭

#### 예제 4: 미완료자 상세 정보
